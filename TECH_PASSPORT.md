## 2. ЛОГИКА АВТОМАТИЗАЦИИ (GitHub Actions)

Проект использует **GitHub Actions** для поддержания данных в актуальном состоянии без ручного вмешательства.

*   **Триггер:** Ежедневно в 03:00 UTC (06:00 МСК) или вручную через `workflow_dispatch`.
*   **Скрипт `update_rates.js`:**
    *   **Модель:** `gemini-2.0-flash`.
    *   **Действие:** Использует Google Search через Gemini AI для поиска актуальной ключевой ставки ЦБ РФ и ставок ТОП-5 банков (Сбер, ВТБ, Альфа и др.).
    *   **Сохранение:** Записывает результаты в `rates.json`.
*   **Скрипт `update_inflation.js`:**
    *   **Модель:** `gemini-2.0-flash`.
    *   **Действие:** Ищет официальные данные Росстата по годовой инфляции.
    *   **Сохранение:** Обновляет `inflation.json`.
*   **Результат:** GitHub Actions автоматически делает `git commit` и `push` обновленных JSON-файлов в репозиторий.

## 3. ВЗАИМОДЕЙСТВИЕ ДАННЫХ

Проект спроектирован как **Serverless Static Web App**:

1.  **Хранение:** Все динамические данные лежат в `JSON` файлах в корне репозитория.
2.  **Доставка:** При открытии страницы браузер выполняет `fetch()` соответствующего JSON-файла.
3.  **Обработка (DOM):**
    *   Скрипты на страницах парсят JSON.
    *   Данные подставляются в инпуты (например, текущая ставка ЦБ в ипотечный калькулятор).
    *   Графики (Chart.js) строятся на лету на основе полученных массивов.
4.  **Пользователь:** Взаимодействует с UI-компонентами (Tailwind CSS), меняет параметры, а `Chart.js` мгновенно перерисовывает результаты.

## 4. ВНЕШНИЕ ЗАВИСИМОСТИ (CDN)

Для работы калькуляторов используются следующие библиотеки через CDN:

*   **Tailwind CSS:** Стилизация интерфейса и адаптивность.
*   **Chart.js:** Визуализация данных (графики платежей, динамика инфляции).
*   **html2canvas:** Позволяет пользователю делать скриншоты результатов для шеринга.
*   **Google Fonts:** Шрифты (Inter/Sans-serif).

## 5. УЯЗВИМОСТИ И ПЛАНЫ

### Технические «костыли» и риски:
*   **API Ключи:** Скрипты автоматизации зависят от `GEMINI_API_KEY`, который должен быть прописан в Secrets репозитория.
*   **Обработка ошибок AI:** Если Gemini вернет некорректный JSON, скрипты упадут (реализованы базовые fallback-значения, но логика очистки данных `cleanNum` может быть улучшена).
*   **Отсутствие Backend:** Весь расчет идет на клиенте. Это безопасно для конфиденциальности, но сложные симуляции могут подтормаживать на слабых устройствах.
*   **Firebase:** Модуль аналитики Firebase был удален для упрощения проекта, так как требовал сложной настройки на стороне пользователя.

### Функциональные пробелы:
*   **Валюты:** Все расчеты жестко привязаны к рублю. Нет поддержки мультивалютности.
*   **Налоги:** В калькуляторе «Рентген Доходов» используется 5-уровневая шкала налогов РФ, но она может потребовать обновления при изменении законодательства.
*   **Оффлайн-режим:** Без интернета (и CDN) калькуляторы не загрузятся.

### Планы по оптимизации:
1.  **Локализация:** Вынос всех строк в отдельный JSON для поддержки других языков.
2.  **PWA:** Добавление Service Worker для работы калькуляторов без интернета.
3.  **Тестирование:** Внедрение автоматических тестов (Playwright/Jest) для проверки математической точности расчетов.
4.  **Унификация AI:** (Выполнено) Все скрипты переведены на единую бесплатную модель `gemini-2.0-flash`.
